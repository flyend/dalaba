K-dimension tree

K-d Tree是一种树结构索引方式，其基本思想是对搜索空间（高维空间）进行层次划分，K-d tree是一种平衡二叉树（BST）。

根据划分的空间是否有混叠可以分为Clipping和Overlapping两种：

  • K-d tree 划分空间没有重叠
  • R Tree 划分空间相互有交叠

维度（dimension）

k-d树是一种空间（维度）划分，将空间划分为特定的几个部分，然后在特定空间的部分内进行相关搜索

二维平面划分

k近邻算法 KNN(k-nearest neighbor)

距离函数（distance）

不同维度有各种不同的距离计算，常用的距离函数有

  • 曼哈顿距离（Manhattan distance）
  • 欧几里得距离（Euclidean distance）
  • 余弦相似度
  • 汉明距离（Hamming distance）

距离函数定义

曼哈顿距离（Manhattan distance）

二维平面两点a(x1,y1)与b(x2,y2)间的曼哈顿距离

两个n维向量a(x11,x12,…,x1n)与 b(x21,x22,…,x2n)间的曼哈顿距离

欧氏距离（Euclidean Distance）

二维平面上两点a(x1,y1)与b(x2,y2)间的欧氏距离

三维空间两点a(x1,y1,z1)与b(x2,y2,z2)间的欧氏距离

构建K-d tree（build tree）

new Dalaba.KDTree([2, 3, 4, 9, 7, 1, 5])

构建K-d tree（build tree）2维

new Dalaba.KDTree([{x: 2, y: 3}, {x: 5, y: 4}, {x: 9, y: 6}, {x: 4, y: 7}, {x: 8, y: 1}, {x: 7, y: 2}], ["x", "y"])

空间划分（split）

  • 1. 维度排序
  • 2. 从中间元素开始建根节点
  • 3. 递归 [0, m) 为左树（小于根节点），递归 [m + 1, m) 为右树（大于根节点）
  • 4. 结束递归条件为 n < 2，符合二叉树结构
  • 5. m为 n / 2，符合平衡二叉树

K-d tree 搜索，近邻选择（nearest）

  • 最小堆（Heap）
  • 二分搜索point与tree node距离
  • 小于堆顶元素则更新堆（push）
  • 取堆的top k元素

最小堆（Heap）

辅助K-d tree中邻近K个元素

  • 根节点最小
  • 每次push更新堆顶元素为最小

堆元素个数大于k则pop，维护k个元素的最小堆

K-d tree 插入（insert）

  • 距离比较
  • 递归插入左节点或右节点
  • 平衡二叉树（rotate left and rotate right）

K-d tree 删除（delete）

  • 无子树，本身为叶子节点，直接删除
  • 有右子树，在t.right寻找切分维度最小的节点p，然后替换被删节点t，递归处理删除节点p
  • 无右子树有左子树，在t.left寻找切分维度最小的节点p，然后用节点p替换被删节点t；将原t.left作为p.right；递归处理删除节点p
  • 寻找切分维度为lg(n)的二叉树最小维度查找

时间复杂度

操作平均复杂度最坏复杂度

删除节点 O(logn) O(n)
最近邻搜索 O(logn) O(n)
